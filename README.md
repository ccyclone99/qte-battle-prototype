# QTE 战斗系统原型

一个基于 HTML5 Canvas 的回合制 QTE 战斗原型，核心机制是「主行动条 + 插入式子 QTE 链」。

## 运行方式

直接用浏览器打开 `index.html` 即可，无需构建工具。

```
双击 index.html
```

或者使用本地服务器：

```bash
node server.js
# 然后访问 http://localhost:8765
```

## 操作说明

### 玩家回合

| 按键 | 武器 | QTE 链 |
|---|---|---|
| `A` | 大剑 | 拔剑 → 蓄力 → 挥砍 |
| `S` | 法杖 | 举杖 → 吟唱（节奏）→ 发动 |
| `D` | 双刀 | 突进 → 连斩一 → 连斩二 → 终结 |

- 先按对应按键切换/确认武器，再按一次进入该武器的 QTE 链。
- QTE 触发后，**主行动条暂停**，弹出独立的子 QTE 判定条。
- 在绿色判定区内按键，黄色细线处为 Perfect 时机。
- 成功完成整条 QTE 链后对敌人造成伤害。

### 敌方回合

敌人会随机使用以下攻击：

| 攻击 | 可响应方式 |
|---|---|
| 直刺 | 闪避 / 弹反（SPACE） |
| 横扫 | 闪避 / 格挡（F） |
| 重砸 | 只能闪避（SPACE） |
| 法术咏唱 | 弹反打断（SPACE） |

- 敌人预警条进入绿色区域后，按对应按键触发防御 QTE。
- 闪避成功可追击；弹反/格挡成功可接反击，并可能眩晕敌人。

## 文件结构

```
.
├── index.html
├── style.css
├── server.js          // 本地 HTTP 服务器（可选）
└── js/
    ├── main.js          // 游戏循环与入口
    ├── battle.js        // 战斗状态机
    ├── qte-runner.js    // QTE 链执行器
    ├── input.js         // 输入缓冲
    ├── renderer.js      // Canvas 渲染
    ├── utils.js         // 工具函数
    └── data/
        ├── weapons.js   // 武器 QTE 链配置
        ├── defenses.js  // 防御 QTE 链配置
        └── enemies.js   // 敌人攻击配置
```

## 设计要点

- **配置驱动**：武器、防御、敌人的行为都通过 JSON/对象配置，便于扩展和调数值。
- **状态机清晰**：`player_turn` / `enemy_turn` / `qte_running` / `resolving` / `game_over` 分离。
- **统一的 QTE 执行器**：敌我双方的 QTE 链共用同一套 `QTEChainRunner`，只需传入不同配置。
- **节奏节点**：法杖吟唱使用拍子系统，错过过多拍子会导致咏唱失败。
